>> What is Context API? (Simple Explanation)
========================================================

   Context API is a way to share data between components
   WITHOUT passing props at every level.

   Why use it?
   Imagine: App ‚Üí Parent ‚Üí Child ‚Üí GrandChild
   If GrandChild needs data from App, you have to pass it
   through Parent and Child even though they DON'T need it.
   This is called "PROP DRILLING" ‚Äî messy and annoying.

   Context API lets GrandChild GRAB the data directly
   from App ‚Äî skipping the middle components.

   Think of it like this:
   ‚Ä¢ Props   = handing a letter person to person in a chain
   ‚Ä¢ Context = putting the letter on a NOTICE BOARD
               ‚Äî anyone can read it directly

   3 simple steps:
   1. CREATE the notice board   ‚Üí React.createContext()
   2. PIN data on it            ‚Üí <Context.Provider value={data}>
   3. READ data from it         ‚Üí useContext(Context) in any component

   That's it. Context API = a shared notice board for
   your React components.

>> What This Project Does:
   - A simple Login + Profile app
   - User types username & password in Login component
   - Profile component shows "Welcome {username}" 
   - Login & Profile are SIBLINGS ‚Äî they share data using Context API
     (no prop drilling needed!)

========================================================
>> THE PROBLEM CONTEXT API SOLVES
========================================================

   Without Context:
     App has data ‚Üí passes to Child1 ‚Üí passes to Child2 ‚Üí passes to Child3
     This is called "PROP DRILLING" ‚Äî ugly and hard to maintain

     App (state lives here)
      ‚îú‚îÄ‚îÄ Login (needs setUser)    ‚Üê pass as prop
      ‚îî‚îÄ‚îÄ Profile (needs user)    ‚Üê pass as prop

   With Context:
     Data sits in a "global store" ‚Üí ANY component can access it directly
     No need to pass props through every level

     UserContext (global store)
      ‚îú‚îÄ‚îÄ Login ‚Üí grabs setUser directly
      ‚îî‚îÄ‚îÄ Profile ‚Üí grabs user directly

========================================================
>> FILES & THEIR ROLES
========================================================

   1. UserContext.js         ‚Üí Creates the Context (the empty box)
   2. UserContextProvider.jsx ‚Üí Fills the box with data + wraps children
   3. Login.jsx              ‚Üí Writes data TO the context (setUser)
   4. Profile.jsx            ‚Üí Reads data FROM the context (user)
   5. App.jsx                ‚Üí Wraps everything inside the Provider

========================================================
>> 3-STEP PROCESS TO USE CONTEXT API
========================================================

   STEP 1: CREATE the Context
   ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
   File: context/UserContext.js

     import React from "react";
     const UserContext = React.createContext();
     export default UserContext;

   - React.createContext() creates an empty context object
   - Think of it as creating an empty container/box
   - This file is very small ‚Äî just creates and exports


   STEP 2: PROVIDE the Context (fill the box with data)
   ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
   File: context/UserContextProvider.jsx

     const UserContextProvider = ({children}) => {
         const [user, setUser] = React.useState(null);
         return (
             <UserContext.Provider value={{user, setUser}}>
                 {children}
             </UserContext.Provider>
         )
     };

   - Creates the actual state (user, setUser)
   - Wraps children inside <UserContext.Provider>
   - value={{user, setUser}} ‚Üí this is what's shared with all children
   - {children} ‚Üí whatever components are wrapped inside it

   Used in App.jsx:
     <UserContextProvider>    ‚Üê everything inside can access user & setUser
       <Login/>
       <Profile/>
     </UserContextProvider>


   STEP 3: CONSUME the Context (take data out of the box)
   ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
   Use useContext() hook in any child component:

   In Login.jsx (WRITES to context):
     const {setUser} = useContext(UserContext);
     // On submit: setUser({username, password})

   In Profile.jsx (READS from context):
     const {user} = useContext(UserContext);
     // Displays: Welcome {user.username}

>> FLOW OF THE APP (Step by Step)
========================================================

   Step 1: App loads
     ‚Üí UserContextProvider wraps Login & Profile
     ‚Üí user = null, setUser = function
     ‚Üí Profile sees user is null ‚Üí shows "please login"

   Step 2: User types "manas" in username, "1234" in password
     ‚Üí Local state in Login updates (useState)

   Step 3: User clicks "Submit"
     ‚Üí handleSubmit runs
     ‚Üí setUser({username: "manas", password: "1234"})
     ‚Üí This updates the CONTEXT state (not just Login's state)

   Step 4: Context state changes ‚Üí React re-renders consumers
     ‚Üí Profile re-renders
     ‚Üí user is now {username: "manas", password: "1234"}
     ‚Üí Shows "Welcome manas"

   KEY INSIGHT: Login never passed any prop to Profile!
   They communicated through the shared Context.

>> IMPORTANT THINGS TO REMEMBER
========================================================

   1. createContext() ‚Üí just creates an empty context
   2. <Context.Provider value={...}> ‚Üí fills it with data
   3. useContext(Context) ‚Üí consumes/reads that data
   4. Any component INSIDE the Provider can access the value
   5. Components OUTSIDE the Provider CANNOT access it
   6. When context value changes ‚Üí ALL consuming components re-render

   Common pattern:
     context/
       UserContext.js          ‚Üê Step 1: Create
       UserContextProvider.jsx ‚Üê Step 2: Provide
     components/
       Login.jsx               ‚Üê Step 3: Consume (write)
       Profile.jsx             ‚Üê Step 3: Consume (read)

========================================================
>> CONTEXT API vs PROPS
========================================================

   | Props                          | Context API                    |
   |--------------------------------|--------------------------------|
   | Pass data parent ‚Üí child       | Share data with ANY component  |
   | Need to drill through levels   | Direct access, no drilling     |
   | Good for 1-2 levels            | Good for deeply nested or      |
   |                                | sibling components             |
   | Simple and explicit            | More setup (3 steps)           |

========================================================
>> ONE-LINE SUMMARY
========================================================

   "Context API = Create a context, Provide it with data at the top,
    and Consume it with useContext() anywhere inside the Provider."
